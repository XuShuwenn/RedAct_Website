RedAct

Redacting Agent Capability Traces for Procedural Skill Protection

RedAct releases auditable agent traces while reducing leakage of reusable procedures: formulas, thresholds, tool choices, validation routines, and recovery strategies.

The tension

Agent traces are audit logs and attack surfaces.

Execution traces show what a deployed agent did: tool calls, intermediate decisions, failures, recovery steps, and final outputs. They are useful for transparency and debugging.

The same traces can expose reusable private skills. A downstream method may recover proprietary formulas, calibrated thresholds, workflow dependencies, and validation routines without access to model weights or original skill files.

Black-box trace disclosure

Raw traces can teach future agents how to repeat protected work.

RedAct treats public traces as security interfaces: useful enough to audit, abstract enough to avoid becoming reusable procedure manuals.

Method

Two layers: selective rewriting and provenance watermarking.

01

Locate protected key information

An owner-side LLM pass inspects the task and skill package to identify formulas, constants, thresholds, tool choices, validation routines, and private heuristics.

02

Review and deduplicate

A lightweight human review removes spurious items and merges duplicates before release-time rewriting.

03

Rewrite the trace

The rewriter abstracts protected procedural details while preserving final outputs, execution order, tool-use evidence, and verifier-critical fields.

04

Add behavioral watermarks

Optional neutral action patterns support downstream provenance analysis when released traces are reused by student systems.

Results

RedAct removes the raw-trace reuse advantage while keeping traces auditable.

Raw traces leak procedural utility

Across extraction, evolution, and retrieval reuse, raw traces recover much of the oracle-skill benefit, reaching 71.8-73.7% average step success.

Protected traces suppress transfer

After RedAct rewriting, the same reuse methods fall to 65.5-67.5% average step success, no higher than the 68.0% no-skill baseline.

Audit evidence remains

Release diagnostics show that final answers, tool names, verifier paths, and schema fields are mostly preserved while key procedural items are removed.
